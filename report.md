# Low Issues

## L-1: Centralization Risk for trusted owners

Contracts have owners with privileged rights to perform admin tasks and need to be trusted to not perform malicious updates or drain funds.

**Found in:** `contracts/ShipmentTracker.sol` [Line 57]

```solidity
function setThreshold(uint256 _newThreshold) public onlyOwner {
```

**Risk:** A single owner can change the safety temperature threshold at any time with no on-chain audit trail, no timelock, and no event emitted. In a medical supply chain context, this is a regulatory compliance risk.

**Fix Applied:**
```solidity
// ✅ Added ThresholdUpdated event for full audit trail
event ThresholdUpdated(uint256 indexed oldThreshold, uint256 indexed newThreshold);

function setThreshold(uint256 _newThreshold) external onlyOwner {
    emit ThresholdUpdated(temperatureThreshold, _newThreshold);
    temperatureThreshold = _newThreshold;
}
```

**Status:** ✅ FIXED — Event added for audit trail.

## L-4: Event is missing `indexed` fields

Index event fields make them more quickly accessible to off-chain tools that parse events. The `temperature` field in `TemperatureAlert` is not indexed.

**Found in:**
- `contracts/ShipmentTracker.sol` [Line 19]
- `contracts/ShipmentTrackerV2.sol` [Line 11]

```solidity
// ❌ temperature field not indexed — slow for frontend event filtering
event TemperatureAlert(uint256 indexed shipmentId, uint256 temperature);
```

**Why this matters:** The frontend uses `useWatchContractEvent` to listen for `TemperatureAlert` events in real time. Indexed fields allow efficient filtering without scanning all logs.

**Fix Applied:**
```solidity
// ✅ Both fields indexed for efficient frontend filtering
event TemperatureAlert(uint256 indexed shipmentId, uint256 indexed temperature);
```

**Status:** ✅ FIXED — `temperature` field now indexed in both contracts.

## L-5: PUSH0 is not supported by all chains

Solc `0.8.20` targets the Shanghai EVM by default, which includes `PUSH0` opcodes. This may cause deployment failures on L2 chains that don't support `PUSH0`.

**Found in:**
- `contracts/ShipmentTracker.sol` [Line 2]
- `contracts/ShipmentTrackerV2.sol` [Line 2]

**Fix Applied** — Added EVM version target in `hardhat.config.js`:
```javascript
module.exports = {
  solidity: {
    version: "0.8.20",
    settings: {
      evmVersion: "paris"  // ✅ paris = pre-Shanghai, no PUSH0 opcodes
    }
  }
};
```

**Status:** ✅ MITIGATED — EVM version set to `paris` in Hardhat config.

# Manual Findings

## M-1: Unprotected `updateStatus` — Missing Access Control

> This finding was identified through manual code review. Aderyn v0.1.9 does not automatically flag missing access control on non-financial state-changing functions.

**File:** `contracts/ShipmentTracker.sol` & `contracts/ShipmentTrackerV2.sol`
**Function:** `updateStatus(uint256, uint256)`

**Description:**
The `updateStatus` function is `public` with no access control modifier. Any external account can submit arbitrary temperature readings for any shipment ID, enabling:
- Injection of fake temperature data by malicious actors
- False `TemperatureAlert` events triggering the red UI overlay on the frontend
- Manipulation of the `shipmentTemperatures` mapping

**Vulnerable Code:**
```solidity
// ❌ Anyone can call — no access control
function updateStatus(uint256 _shipmentId, uint256 _currentTemp) public {
    if (_currentTemp > temperatureThreshold) {
        emit TemperatureAlert(_shipmentId, _currentTemp);
        revert("Temperature breach: Vaccine safety threshold exceeded");
    }
    shipmentTemperatures[_shipmentId] = _currentTemp;
}
```

**Fix Applied:**
```solidity
// ✅ Restricted to owner (authorized IoT sensor wallet)
function updateStatus(uint256 _shipmentId, uint256 _currentTemp) external onlyOwner {
    if (_currentTemp > temperatureThreshold) {
        emit TemperatureAlert(_shipmentId, _currentTemp);
        revert("Temperature breach: Vaccine safety threshold exceeded");
    }
    shipmentTemperatures[_shipmentId] = _currentTemp;
}
```

**Status:** ✅ FIXED — `onlyOwner` added to `updateStatus` in both V1 and V2.
